# 认证历程Adobe [!DNL Workfront Fusion] 开发人员专业人员

了解如何成为认证的Adobe Developer Professional。

## 最低体验

您应该：

* 使用 [!DNL Workfront] API和API资源管理器，但不一定精通API调用
* 能够使用自定义API调用处理 [!DNL Workfront] 和其他API
* 能够将模块用于 [!DNL Workfront] 应用程序和其他操作（如创建、读取、更新、删除和搜索）
* 能够构建能够产生业务成果的可行解决方案，而不影响业务成果 [!DNL Fusion] 护栏
* 能够选择适当的触发器模块以实现业务成果
* 了解如何使用流量控制（路由、迭代器/聚合器、解释流量）
* 能够进行基本故障排除和错误处理
* 有效测试和启用情景的功能
* 能够管理组织团队和用户
* 了解通用连接器在REST API Oauth2中的用途和用法

>[!NOTE]
>
>您的证书有效期为2年，从您通过之日算起。 我们建议尽可能使用较新版本，因为它涵盖较新的产品功能。

**目标受众：**

* 自动化工程师开发人员
* 自动化、流程优化和自动化管理器
* 集成顾问、顾问、合作伙伴
* 系统集成商/合作伙伴
* 业务分析员、应用业务分析员
* 系统管理员（IT、MIS、技术、知识系统、AI管理员）

**考试详细信息：**

* 级别：专业（0-12个月经验）
* 通过分数：33/51
* 时间：102分钟
* 投放：在线监控（需要摄像头访问）或测试中心监控
* 可用语言：英语
* 成本：125美元（全球）/ 95美元（印度）
* 考试ID:AD0-E902

**考试目标和范围**

第1节：基本技术概念(39%)

* 给定 [!DNL Fusion] 需要采用其他字段格式的场景，提供可用于正确转换数据的可能函数
* 给定 [!DNL Fusion] 需要将函数嵌套在另一个函数中的情况，请选择格式正确的表达式
* 确定使用团队控制对连接和其他 [!DNL Fusion] 功能
* 给定 [!DNL Fusion] 方案，识别系统和功能限制
* 给定 [!DNL Fusion] 需要时区处理、区分用户、可见组织以及需要覆盖时区的位置的方案
* 给定 [!DNL Fusion] 方案中，选择相应的 [!DNL Workfront] 模块或操作
* 给定 [!DNL Fusion] 方案中，选择将相应转换输入数据以输出数据的映射面板表达式和/或模块
* 确定正确的使用方法 [!DNL Fusion] 开发工具，用于排除执行中的错误或确定对第三方系统发出的调用和响应
* 给定 [!DNL Fusion] 在另一路径中需要一条路径中的数据的情况下，提供集成解决方案，以使该数据在其他路径上可用
* 确定使用或生成 [!DNL Fusion] 业务需求模板
* 给定 [!DNL Fusion] 方案中，确定要执行的操作，以查看执行和/或解决错误
* 在 [!DNL Fusion] 方案设置以满足方案需求
* 给定 [!DNL Fusion] 方案中，选择适当的流量控制和/或确定流量控制模块的正确组合

第二节：情景设计和架构(35%)

* 给定一系列步骤，选择要解析JSON并转换为包的正确序列
* 给定一组要求，确定正确的数据查找方法
* 识别即时触发器与计划触发器之间的差异
* 确定将文档上传到的正确模块 [!DNL Workfront]
* 在满足业务需求的情况下，确定正确的跟踪方法 [!DNL Fusion] 处理了记录（不是用户）和时间
* 给定 [!DNL Fusion] 方案中，确定简化设计和/或优化维护的机会
* 识别 [!DNL Fusion] 记录方案的最佳实践
* 在某种情况下，确定通过某种情景减少数据流的机会
* 给出搜索模块和后续聚合器上显示的包数量示例，解释包检查器处理和显示的操作数量的变化
* 识别在包检查器中找到的信息
* 描述了需要将方案存档超过60天并选择唯一可能的方式（导入/导出Blueprint）的情况
* 给出错误403的示例：禁止，请正确选择错误消息的来源和原因

第三节：测试和错误处理(16%)

* 确定测试计划的元素
* 识别测试用例的元素
* 给定 [!DNL Fusion] 方案中，正确确定处理不可靠服务的适当指令
* 识别在 [!DNL Fusion] 方案
* 确定如何跟踪、读取和解决未完成的执行
* 给定 [!DNL Fusion] 缺少所需数据的方案中，选择处理无效数据的方法

第四节：使用API(10%)

* 如果是返回429的第三方API:请求过多，请确定一个解决方案以防止或处理错误
* 给定 [!DNL Fusion] 新功能在 [!DNL Workfront] 模块，识别正确的引用和模块类型以使用新功能
* 如果第三方系统没有专用应用程序，请识别HTTP应用程序并选择相应的模块
* 识别CRUD操作与REST API的其他常见功能之间的差异
