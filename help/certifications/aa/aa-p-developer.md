# 的认证历程 [!DNL Adobe Analytics] 开发人员专业版

了解如何成为认证 [!DNL Analytics] 开发人员专业版。

## 最低体验

您应具有：

* 6-12个月的使用经验 [!DNL Adobe Analytics] 实施（Web和移动应用程序）
* 根据分析师提供的要求，作为支持Analytics实施团队在IT和业务之间架起桥梁，了解业务需求如何转化为技术解决方案(SDR)的参与者的经验
* 了解需要哪种类型的变量、在何处以及如何收集数据、设计数据模型（数据层）、配置和维护标签管理解决方案

体验以下任务：

* 解释解决方案设计文档(SDR)并理解所有内容的目的 [!DNL Adobe Analytics] 收集的变量
* 了解和使用Adobe(Experience Platform)Launch
* 了解基线的配置 [!DNL Adobe Analytics] 维度和事件页面名称、链接名称、Activity Map等，但将与专家开发人员合作，实现更多客户实施 — 变量映射仍由专家级别开发人员拥有
* 按照AA专家提供的设计在Launch中创建和配置规则
* 测试和验证在变量和事件触发中捕获的数据，利用Web控制台调试器或Charles日志等移动应用程序系统
* 了解数据层是什么，并了解如何读取JavaScript对象
* 共享报告、区段和请求访问的过程，但不包括如何管理用户组、治理协议以及管理SDR
* 提供指导并配置移动/SDK解决方案
* 了解视频跟踪（基于事件）背后的通用框架（给定知识库文档）
* 从Analysis Workspace执行基本报表，并了解 [!DNL Adobe Analytics] data warehouse
* Adobe Experience Platform Launch的基础体验(Adobe发布)

>[!NOTE]
>
>您的认证从您通过之日起有效期为两年。 我们建议尽可能使用较新的版本，因为它涵盖了较新的产品功能。

>[!BEGINTABS]

>[!TAB 准备就绪]

**目标受众：**

* 实施专家/工程师
* 开发者
* 架构师

**考试详细信息：**

* 级别：专业（0至12个月经验）
* 及格分数：31/50
* 时间：100分钟
* 投放：在线监护（需要访问摄像头）
* 可用语言：英语
* 费用：125美元（全球）和95美元（印度）
* 考试ID：AD0-E213

**考试目标和范围**

第1部分：Adobe Experience Cloud生态系统中的Analytics (14%)

* 识别Adobe Experience Cloud ID的特性和功能
* 确定启动Adobe的用途
* 了解如何启用/支持 [!DNL Adobe Analytics] Adobe Experience Cloud生态系统中的标记

第2部分：基于解决方案设计参考的Analytics策略和设计(12%)

* 提供解决方案设计参考，确定必须填充哪些数据对象
* 阅读技术规格文档的一部分，确定如何实施数据捕获

第3部分：Analytics实施和配置(30%)

* 确定要部署的步骤 [!DNL Adobe Analytics] 代码
* 确定用于Analytics跟踪的最小部署代码的功能
* 确定在中配置各种组件的步骤 [!DNL Adobe Analytics] UI
* 识别的功能 [!DNL Adobe Analytics] 报表包和变量设置
* 识别数据层和数据层值的特性
* 识别用于跟踪功能（如链接跟踪和电子商务跟踪）的应用程序测量函数

第4节：标签管理系统(18%)

* 使用标记审核确定是否已满足要求
* 确定使用AdobeLaunch配置网站标记的步骤
* 给定一个场景，请确定如何在AdobeLaunch中启用、修改扩展并对其进行故障排除
* 确定如何创建规则和数据元素

第5部分：Mobile Services和API (8%)

* 确定的功能 [!DNL Adobe Analytics] API（数据馈送、仓库、数据源、报表）
* 确定处理规则和移动服务的特征

第6部分：测试、验证和故障排除(18%)

* 确定常见JavaScript错误的含义
* 确定共同点的含义 [!DNL Adobe Analytics] 变量错误
* 给定场景，确定应遵循的测试和调试验证最佳实践
* 识别如何跟踪 [!DNL Adobe Analytics] 服务器调用在其生命周期内（即从信标请求到显示在报表中）

>[!TAB 准备好]

您不必在参加考试前完成培训，仅靠培训无法为您提供通过考试所需的知识和技能。 培训与成功的在职经验相结合，对于为您提供通过考试所需的存储库至关重要。
